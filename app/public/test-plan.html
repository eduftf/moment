Moment — Test Plan

Comprehensive test plan for the Moment Zoom App covering authorization, scopes, and all functionality.

Overview

Moment is a Zoom App that captures screenshots during meetings. It consists of two components:

  1. Zoom App — a sidebar panel that runs inside Zoom meetings (iframe via @zoom/appssdk), listens for meeting events, and sends capture commands.
  2. Companion — a lightweight desktop application that runs locally on the user's computer, receives capture commands via WebSocket, and takes screenshots using the OS-native screenshot tool.

App URL: https://moment.gtools.space

Documentation: https://moment.gtools.space/docs

Companion Download: GitHub Releases

Prerequisites

  • A Zoom account with the Moment app installed
  • The Moment Companion app downloaded and running on your computer
  • A Zoom meeting with at least one other participant (for reaction/peak triggers)

To test the app:

  1. Install Moment from the Zoom Marketplace listing
  2. Download the Companion from GitHub Releases
  3. No additional account or login is required — Moment uses Zoom's built-in OAuth

1. Authorization Flow

1.1 App Installation

  1. Go to the Moment listing on Zoom Marketplace
  2. Click Add
  3. Review the requested permissions (in-meeting access)
  4. Click Authorize
Expected: App is added to your Zoom account. You see a confirmation page.

1.2 OAuth Callback

  1. After authorizing, the browser redirects to the OAuth callback URL
Expected: The callback page loads successfully at moment.gtools.space/auth/callback

1.3 Opening the App in a Meeting

  1. Start or join a Zoom meeting
  2. Click the Apps button in the bottom meeting toolbar
  3. Select Moment from the apps list
Expected: The Moment sidebar panel opens on the right side of the meeting window. The Zoom SDK initializes and the sidebar shows connection status.

3. Functionality Testing

Companion Connection

3.1 Companion Connection

  1. Ensure the Companion app is running on your computer
  2. Open Moment in a Zoom meeting
Expected: Status bar shows "Companion: Connected". The companion auto-discovers via WebSocket on localhost:54321.

3.2 Companion Disconnection

  1. Open Moment in a meeting with the Companion running
  2. Quit the Companion app
Expected: Status bar changes to "Companion: Not running". After approximately 3 seconds, a setup overlay appears suggesting to download the Companion.

3.3 Companion Reconnection

  1. With Moment open and Companion disconnected, restart the Companion
Expected: Status bar updates to "Companion: Connected" within 5 seconds (auto-reconnect interval).

Reaction Capture

3.4 Reaction Capture (Automatic Trigger)

  1. Open Moment in a meeting with Companion connected
  2. Ensure the "Reaction Capture" toggle is enabled in the sidebar
  3. Have a participant send an emoji reaction (e.g., thumbs up, clap, heart)
Expected: A screenshot is captured. A new moment appears in the Moments list in the sidebar with the reaction emoji, trigger type "reaction", and timestamp. The screenshot PNG file is saved to ~/Moment/{meeting-topic}-{date}/images/.

3.5 Reaction Debounce

  1. Send multiple emoji reactions rapidly (within 5 seconds)
Expected: Only one screenshot is captured per 5-second window. Subsequent reactions within the cooldown are ignored.

3.6 Reaction Emoji Filter

  1. Open trigger settings and select specific reactions (e.g., only thumbs up)
  2. Send a heart reaction
  3. Send a thumbs up reaction
Expected: No screenshot for heart (not in allowed list). Screenshot captured for thumbs up.

Peak Detection

3.7 Peak Detection (Automatic Trigger)

  1. Open Moment in a meeting with Companion connected
  2. Ensure the "Peak Detection" toggle is enabled
  3. Start with 2 participants. Have a 3rd participant join.
Expected: Participant count increases. When count exceeds previous maximum, a screenshot is captured with trigger type "peak".

3.8 Peak Detection Cooldown

  1. After a peak trigger fires, have another participant join within 2 minutes
Expected: The new peak is recorded, but no screenshot is captured until the 2-minute cooldown expires.

Manual Capture

3.9 Manual Capture

  1. Open Moment in a meeting with Companion connected
  2. Click the Capture button in the sidebar
Expected: A screenshot is captured immediately. A new moment appears in the list with trigger type "manual".

Moments List & Archive

3.10 Moments List

  1. Capture several screenshots using different triggers
Expected: All moments are listed chronologically in the sidebar. Each shows a thumbnail, trigger type (reaction/peak/manual), and timestamp.

3.11 Delete a Moment

  1. In the Moments list, click the delete button on a captured moment
Expected: The moment is removed from the list and the corresponding PNG file is deleted from disk.

3.12 Meeting Archive

  1. Open Moment in a meeting. Capture a few screenshots. End the meeting.
  2. Navigate to ~/Moment/ on your computer.
Expected: A folder named {meeting-topic}-{YYYY-MM-DD}/ containing: archive.html (timeline + gallery viewer), archive.json (raw data), and images/ folder with PNG screenshots.

3.13 Archive HTML Viewer

  1. Open the archive.html file in a web browser
Expected: A self-contained page with meeting header, participant timeline, event list, and screenshot gallery.

Settings

3.14 Settings — Save Location

  1. Open Settings in the sidebar, change the save location
  2. Capture a screenshot
Expected: Screenshot saved to the new custom folder.

3.15 Settings — Capture Mode

  1. Open Settings, change capture mode (Window / Screen / Video with margins)
  2. Capture a screenshot
Expected: Screenshot reflects the selected capture mode.

Companion Onboarding

3.16 Companion Onboarding

  1. Open Moment in a meeting without the Companion running
  2. Wait 3 seconds
Expected: An onboarding overlay appears with download links for the Companion app.

3.17 Cross-Platform Screenshot

  1. macOS: Run Companion and capture → uses native screencapture
  2. Windows: Run Companion and capture → uses PowerShell screen capture
Expected: Screenshots captured correctly on each platform.

4. Privacy and Security

4.1 Local-Only Data

  1. Capture screenshots during a meeting
  2. Monitor network traffic
Expected: No screenshot data is sent to any external server. All data stays on the local machine. The Companion only communicates via localhost:54321.

4.2 WebSocket Security

  1. The Companion WebSocket server only accepts connections from localhost
Expected: External connections to port 54321 are rejected.

5. Edge Cases

5.1 No Companion Running

  1. Open Moment without the Companion installed or running
  2. Try to use reaction/peak/manual triggers
Expected: Sidebar shows "Companion: Not running" and an onboarding overlay. Triggers fire but moments are marked as "Missed" (no screenshot file saved).

5.2 Meeting with One Participant

  1. Open Moment in a meeting where you are the only participant
Expected: Peak detection shows 1 participant. Reaction trigger and manual capture work normally.

5.3 Long Meeting

  1. Run Moment for an extended meeting (30+ minutes)
Expected: All triggers continue to work. Archive accumulates events and screenshots without issues.
